# Шаг 1. Настройка CLAUDE.md

Это первый и единственный шаг этого файла: собрать у себя в корне папки `Система/` главный файл-проводник `CLAUDE.md`. Больше ничего в этом шаге не делаем — ни проектных папок, ни обзоров. Только один файл.

---

## Зачем нужен CLAUDE.md

`CLAUDE.md` — это первое, что нейросеть читает при каждом запуске. Туда вы один раз кладёте контекст, который иначе пришлось бы повторять в каждом чате: кто вы, какие у вас проекты, как с вами общаться, что нельзя делать, куда смотреть под каждый тип запроса.

После того как этот файл собран:
- ИИ перестаёт задавать одни и те же вопросы каждый раз.
- Отвечает в вашем стиле, не общими словами.
- Знает имена, проекты, ограничения — не выдумывает.
- Понимает, какой файл открыть под конкретный запрос.

---

## Как устроен промпт-помощник

Это интервью **по одному вопросу за раз**. ИИ задаёт ровно один вопрос → ждёт ответа → задаёт следующий. Не вываливает блоки по 5 вопросов.

К большинству вопросов есть **варианты ответа** — выбираете цифру или пишете свой. Где варианты не нужны (имя, должность, ситуация) — просто пишете текстом.

Промпт **адаптивный**: ветка вопросов зависит от того, кто вы. Владельцу бизнеса задаются одни вопросы (проекты, команда, NSM), сотруднику — другие (должность, обязанности, регламенты), студенту — третьи. Лишнего не спросят.

Технические штуки (путь к папке, формат мусорки, правила удаления) ИИ выставляет **по умолчанию сам**. Вас спрашивает только в конце: «оставить дефолты или что-то поменять».

---

## Как запустить

1. Откройте Claude Code в папке, которую назначили будущей `Система/`. Папка может быть пустой.
2. Скопируйте всё содержимое блока `START PROMPT` … `END PROMPT` ниже.
3. Вставьте в Claude Code как обычное сообщение.
4. Отвечайте на вопросы по одному.
5. После последнего вопроса ИИ покажет сводку «что я понял» и спросит «создавать CLAUDE.md?». Отвечаете «да» — он пишет файл в корень.

Времени уйдёт 25–45 минут. Можно делать с перерывами — состояние держится в чате.

## Раздел 8. Правила работы

Правила обязательны для ИИ и для пользователя. ИИ читает их при каждом запуске и не имеет права нарушать без явного разрешения.

### 8.1. Базовые запреты

- Все новые файлы только внутри корня `Система/`.
- Не запускать файлы из `~/Downloads`.
- Не трогать `.env`, `credentials`, `LaunchAgents`, любые файлы с паролями и токенами.
- Не пушить в публичные репозитории, не загружать в облака без явной просьбы.

### 8.2. Удаление файлов — строгое правило

**Запрещено** удалять файлы любыми способами (`rm`, `git rm`, `Edit` с пустым содержимым, перетаскивание в системную корзину) без явного разрешения владельца. Никаких `rm -rf`. Никаких «временных» удалений.

Корректный процесс «удаления»:

1. ИИ спрашивает: «Перенести файл/папку X в Мусорку? Причина: …»
2. После «да» — `mv` в `Хранилище/00_Мусорка/{категория}_{YYYY-MM-DD}/`.
3. При переносе сохраняется исходная структура пути внутри Мусорки (например, файл из `raw/99_Остальное/X.md` идёт в `raw/00_Мусорка/<категория>/99_Остальное/X.md`).
4. После каждого `mv` — запись в `Хранилище/00_Мусорка/_log.md` в формате: `| дата | источник | цель | причина | кто перенёс |`.
5. Из Мусорки ничего не удаляется автоматически. Файлы оттуда удаляет только владелец вручную, когда сочтёт нужным.

Если ИИ не уверен — должен спросить. Лучше один лишний вопрос, чем потерянный файл.

### 8.3. raw — неизменяемое хранилище

Когда появится подпапка `Хранилище/raw/` (после шага 2 курса):

- **Содержимое** файлов в `raw/` никогда не изменяется. Это первоисточники.
- Можно: добавлять новые файлы, переименовывать папки, создавать подпапки для структуры.
- Исключения — папки, которые редактируются: `raw/08_Личное/`, `raw/95_Текущие_задачи/`, `raw/00_Мусорка/`. Только здесь содержимое можно править.
- При конфликте `wiki` и `raw` — верим `raw`. После сверки обновляем `wiki`, не наоборот.

### 8.4. Создание папок

- В корне `Хранилище/` папки нумеруются: `01_`, `02_`, …, `08_`, `95_`, `99_`. Нумерация = приоритет, не дата.
- **Новая подпапка создаётся только если в ней будет минимум 10 файлов.** Меньше — оставляем файлы в текущей папке и описываем их в `README.md` родительской папки.
- В каждой подпапке (любой) обязателен `README.md` — перечисление содержимого, если нет более глубоких подпапок.

### 8.5. README.md в каждой подпапке — обязательно

- В **каждой** проектной и сферной подпапке должен быть `README.md`.
- Первая строка `README.md` — напоминание ИИ:
  `> Перед задачей — прочитай <АБСОЛЮТНЫЙ_ПУТЬ>/CLAUDE.md`
- Дальше — короткое описание: статус, суть, таблица «что куда класть».
- Без этой строки ИИ при работе из подпапки может не открыть корневой CLAUDE.md и потерять контекст.

### 8.6. Именование файлов

- Транскрипты, встречи, заметки: `YYYY-MM-DD_короткое_имя.md`. Дата сортирует хронологически.
- Документы, договоры, регламенты: `<тип>_<контрагент_или_тема>.md`.
- Архивные папки по дате: `<категория>_YYYY-MM-DD/`.
- Без пробелов в именах. Только `_` или `-`.
- Без символов `/`, `\`, `:`, `*`, `?`, `<`, `>`, `|`, `"`.

### 8.7. Журнал операций

Все значимые операции ИИ над структурой фиксирует в `Хранилище/wiki/log.md` (появится после шага 3 курса) или `Хранилище/00_Мусорка/_log.md`:

- Перенос файла в Мусорку → запись в `_log.md` Мусорки.
- Создание / переименование / реструктуризация папок → запись в `wiki/log.md`.
- Обновление wiki-страницы по новому источнику → запись в `wiki/log.md`.

Формат записи: `| дата | действие | объект | причина |`.

### 8.8. Когда ИИ должен спросить, а не сделать сам

- Перед удалением / переносом любого файла.
- Перед изменением содержимого файлов в `raw/` (кроме исключений 8.3).
- Перед созданием новой подпапки, если непонятно, релевантна ли она.
- Перед массовой реструктуризацией (5+ файлов).
- Когда два источника противоречат друг другу — спросить, какой считать актуальным.

## После завершения промпта

1. Проверьте, что `CLAUDE.md` лежит в корне `Система/` (рядом с этим файлом курса).
2. Откройте его в любом редакторе и пробегитесь глазами. Это ваш главный файл — он должен читаться как «правда о вас».
3. Если что-то не так — попросите Claude в том же чате: «открой CLAUDE.md и поправь раздел X».
4. Переходите к файлу `02_Наполнение_Хранилища.md` — там собираем папку с материалами проектов.
