# Шаг 2. Наполнение Хранилища

`CLAUDE.md` уже лежит в корне `Система/`. Теперь нужно создать рядом папку `Хранилище/` и максимально наполнить её материалами по вашим проектам и личной сфере. Чем больше в ней файлов — тем лучше ИИ вас знает.

---

## Главный принцип шага

**Записывайте всё, что происходит в вашей жизни.**

Звонки, встречи, голосовые мысли, переписки, документы, идеи, договоры, ТЗ, регламенты — складывайте сюда оригиналы как есть. Не фильтруйте на входе. Фильтрует ИИ при чтении. Через месяц он будет знать о ваших проектах всё, что нужно для точных ответов.

Минимальная цель этого шага — **20+ файлов** в `Хранилище/`, распределённых по проектным подпапкам. Чем больше реальных материалов вы положите сейчас, тем меньше нужно будет объяснять ИИ в каждом чате.

---

## Структура папки Хранилище/

```
Система/
├── CLAUDE.md                  ← создан на шаге 1
└── Хранилище/                 ← создаём сейчас
    ├── README.md              мастер-индекс подпапок
    ├── 00_Мусорка/            «удалённые» через mv по разрешению
    │   └── _log.md            журнал переносов
    ├── 01_<проект_1>/         первый проект (нумерация = приоритет)
    │   ├── README.md
    │   ├── Заметки_чат.md
    │   ├── 00_Inbox/
    │   ├── Встречи/
    │   └── Документы/
    ├── 02_<проект_2>/
    │   └── ...
    ├── 08_Личное/             личные данные — редактируется
    │   ├── README.md
    │   ├── Здоровье/
    │   ├── Обучение/
    │   ├── Хобби/
    │   └── Распределение/
    ├── 95_Текущие_задачи/     активные задачи — редактируется
    │   └── README.md
    └── 99_Остальное/          буфер несортированного
        └── README.md
```

### Нумерация подпапок

Цифра в начале — это **приоритет / порядок навигации**, не дата создания.

| Префикс | Значение |
|---------|----------|
| `00_` | Служебное: Мусорка |
| `01_` … `08_` | Проекты и крупные сферы в порядке важности |
| `08_Личное/` | По соглашению — личное ближе к концу |
| `95_` | Активные задачи, не привязанные к конкретному проекту |
| `99_` | Хвост: «остальное», буфер |

Если проектов меньше — пропускайте номера. Можно начать с `01_`, `02_`, `03_` и сразу `08_Личное/`, `95_Текущие_задачи/`, `99_Остальное/`.

## Обязательные файлы в каждой проектной подпапке

В **каждой** подпапке `Хранилище/0X_<проект>/` должны лежать два служебных файла. Без них ИИ потеряется в материалах.

### README.md

Короткий путеводитель по папке. Первая строка — напоминание про корневой `CLAUDE.md`, потому что ИИ не всегда сам открывает корень при работе из подпапки.

```markdown
> Перед задачей — прочитай <АБСОЛЮТНЫЙ_ПУТЬ>/CLAUDE.md

# <Название проекта>

**Статус:** активный / запуск / архив
**Суть:** <одна строка о проекте>
**Команда:** <имена + роли или «работаю один»>

## Подпапки

| Папка | Что внутри |
|-------|------------|
| 00_Inbox/ | Новые материалы до сортировки |
| Встречи/ | Транскрипты звонков и встреч (YYYY-MM-DD_имя.md) |
| Документы/ | ТЗ, договоры, регламенты |

## Куда что класть

- Транскрипт нового звонка → `Встречи/YYYY-MM-DD_имя.md`
- Новый договор / ТЗ → `Документы/`
- Не знаю куда → `00_Inbox/`, потом разберём
```

### Заметки_чат.md

Хронологический журнал. Сюда сваливаются короткие записи из диалогов с ИИ, решения, дедлайны, договорённости. Свежее — сверху.

```markdown
# Заметки по проекту <Название>

Журнал решений, договорённостей и операционных деталей. Свежее — сверху.

---

## 2026-05-12

- Договорились с клиентом, что дедлайн сдвигается на 20.06.
- Согласован формат отчёта: одна страница, KPI таблицей.

## 2026-05-10

- Старт проекта. Команда: …
```

---

## Правило: README.md в каждой созданной подпапке

Это критически важно. Любая папка внутри `Хранилище/`, которую вы создаёте — должна иметь свой `README.md`. Даже если внутри только два файла.

Зачем:
- ИИ открывает `README.md` подпапки первым и сразу понимает контекст.
- Через 3 месяца вы сами не вспомните, что внутри `Встречи/01_Старые/`. README напомнит.
- При передаче проекта команде README — это карта.

Минимальное содержимое `README.md` подпапки:

```markdown
> Перед задачей — прочитай <АБСОЛЮТНЫЙ_ПУТЬ>/CLAUDE.md

# <Название подпапки>

**Что внутри:** <одна строка>

## Что класть

- <тип материала> → файл `YYYY-MM-DD_имя.md`
- <тип материала> → подпапка `<имя>/`
```

## Что класть в каждую папку

### Проектные подпапки (`01_<проект>/`, `02_<проект>/`, …)

| Что | Куда |
|-----|------|
| Транскрипт звонка / встречи | `Встречи/YYYY-MM-DD_имя.md` |
| Договор, ТЗ, регламент | `Документы/` |
| Скриншоты переписок | `Документы/` или отдельная `Переписки/` |
| Анализ конкурентов, исследования | отдельная подпапка `Исследования/` или `Маркетинг/` |
| Новый материал, не знаю куда | `00_Inbox/` |
| Решения и договорённости | дописать в `Заметки_чат.md` |

### `08_Личное/`

Это единственная папка, которая **редактируется** свободно, а не только дополняется.

```
08_Личное/
├── README.md
├── Распределение/    все новые личные файлы сначала сюда
├── Здоровье/         диагнозы, анализы, тренировки
│   └── README.md
├── Обучение/         курсы, конспекты
│   └── README.md
├── Хобби/            списки книг / кино / музыки
│   └── README.md
├── Путешествия/
└── Спорт/
```

Что важно для ИИ:
- В `Здоровье/` — учитывается при рекомендациях по спорту / диете / путешествиям.
- В `Хобби/` — списки прочитанного / просмотренного, чтобы ИИ не предлагал повторно.

### `95_Текущие_задачи/`

Рабочая папка под активные задачи, не привязанные к конкретному проекту прямо сейчас. Когда задача закрыта — переносите файлы в соответствующий проект (`0X_<проект>/Документы/` или `Встречи/`).

### `00_Мусорка/`

«Удалённые» файлы. Никаких `rm` — только `mv` сюда. В корне Мусорки лежит `_log.md` — журнал переносов. После каждого `mv` дописываете строку: дата / источник / цель / причина.

### `99_Остальное/`

Буфер. Сюда складываются файлы, для которых пока нет проектной папки. Раз в месяц перебирать и распределять.

---

## Правила именования файлов

| Что | Как | Пример |
|-----|-----|--------|
| Транскрипт встречи / звонка | `YYYY-MM-DD_короткое_имя.md` | `2026-05-12_созвон_клиент.md` |
| Документ (договор, ТЗ) | `<тип>_<контрагент_или_тема>.md` | `договор_клиент_апрель.md` |
| Заметка / Inbox | `YYYY-MM-DD_заметка.md` | `2026-05-12_идея_лендинг.md` |
| Архивная папка по дате | `<категория>_YYYY-MM-DD/` | `templates_2026-05-12/` |

Жёсткие правила:
- Дата в формате `YYYY-MM-DD` — так файлы сортируются хронологически.
- Без пробелов в именах. Только `_` или `-`.
- Без символов `/`, `\`, `:`, `*`, `?`, `<`, `>`, `|`, `"`.
- Один файл = одна единица контекста. Не сливайте две встречи в один файл.

## Источники, откуда брать материалы

Заполнять `Хранилище/` нужно не из головы, а из того, что у вас уже есть. Пробегитесь по этим источникам и сложите файлы в нужные подпапки:

| Источник | Что вытащить |
|----------|--------------|
| Записи звонков (Zoom, Telegram) | Транскрипты → `Встречи/` |
| Голосовые в мессенджерах | Транскрипты → `Встречи/` или `00_Inbox/` |
| Email / переписки с клиентами | Скриншоты или копии → `Документы/` |
| Договоры, ТЗ, регламенты | Оригиналы → `Документы/` |
| Заметки в телефоне / Apple Notes | Перенести → `00_Inbox/` |
| Папки в Notion / Drive / Dropbox | Скачать и переложить → соответствующая проектная папка |
| Медицинские анализы | → `08_Личное/Здоровье/` |
| Списки прочитанных книг / фильмов | → `08_Личное/Хобби/` |

### Как делать транскрипты

Голос превращается в текст быстро — есть несколько вариантов:

- **Whisper** (бесплатная локальная модель OpenAI). Установить можно через ту же сессию Claude Code — попросите ИИ: «помоги установить Whisper и сделать скрипт для транскрибации». Он шаг за шагом проведёт.
- **Веб-сервисы** (например, Буквица для русского) — загрузить файл, получить текст. Лимиты на бесплатном тарифе.
- **Handy** или подобные приложения — горячая клавиша → диктовка → текст вставляется в поле.

Результат одинаковый: получаете `.txt` или `.md` файл → переименовываете по формату `YYYY-MM-DD_имя.md` → кладёте в `Встречи/` нужного проекта.

## Шаблон Хранилище/README.md (мастер-индекс)

Положите это в корень `Хранилище/`. Курсант или ИИ открывает его, когда не знает, в какую подпапку лезть.

```markdown
# Хранилище — карта подпапок

**Назначение:** все исходные материалы (документы, конспекты, файлы) проектов и личных сфер. Каждая подпапка имеет свой README.

## Карта подпапок

| Папка | Статус | Что внутри |
|-------|--------|------------|
| `00_Мусорка/` | служебная | «Удалённые» файлы (через mv). См. `_log.md` |
| `01_<проект_1>/` | активный | <одна строка о проекте> |
| `02_<проект_2>/` | активный | <одна строка о проекте> |
| `08_Личное/` | активный, редактируется | Здоровье, Обучение, Хобби, Спорт |
| `95_Текущие_задачи/` | рабочая, редактируется | Активные задачи без привязки к проекту |
| `99_Остальное/` | буфер | Несортированное |

## Быстрая навигация

1. Это про **проект (бизнес-кейс)** → подпапки `01_*` … `06_*`
2. Это про **меня лично** → `08_Личное/`
3. Это **сейчас в работе** → `95_Текущие_задачи/`
4. Это **новое и неясно куда** → `99_Остальное/`

---

**Дата обновления:** YYYY-MM-DD
```
