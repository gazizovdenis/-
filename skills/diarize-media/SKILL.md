# diarize-media

Этот skill разделяет транскрипт по голосам (спикерам) с помощью pyannote.audio.

## Что делает
Когда пользователь просит разделить запись по голосам / спикерам, ты должен:

1. Принять путь к аудиофайлу.
2. Найти рядом `_transcript.md` (если есть).
3. Запустить скрипт `scripts/diarize.py`.
4. Результат сохранится как `_diarized.md` рядом с транскриптом.
5. Показать пользователю путь к файлу и напомнить переименовать SPEAKER_00/01/... в реальные имена.

## Команда запуска

```bash
python "c:\Users\Denis\Desktop\Система\scripts\diarize.py" "<путь_к_аудио>" --token "<hf_token>"
```

Если известно число спикеров, добавь `--speakers N` — точность выше.

## HuggingFace токен

Токен хранится в переменной окружения `HF_TOKEN` или передаётся через `--token`.
Пользователь предоставил токен: использовать его при запуске.

**Важно**: перед первым использованием модели pyannote/speaker-diarization-3.1
пользователь должен принять условия на HuggingFace:
https://huggingface.co/pyannote/speaker-diarization-3.1

## Куда сохраняется результат

Рядом с транскриптом, с суффиксом `_diarized.md`:
- Транскрипт: `Хранилище/raw/.../2026-05-18_трекинг гипотез_transcript.md`
- Результат:  `Хранилище/raw/.../2026-05-18_трекинг гипотез_diarized.md`

## После диаризации

Файл `_diarized.md` содержит метки вида `SPEAKER_00`, `SPEAKER_01`.
Пользователь вручную меняет на реальные имена (Денис, Анатолий, Азамат, Андрей...).

## Правила

- Не изменяй исходный аудиофайл и транскрипт.
- Если нет транскрипта — создаёт только таймлайн спикеров без текста.
- Если нет pyannote.audio: `pip install pyannote.audio`.
