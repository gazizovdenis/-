# answer-checker

Скилл для проверки качества ответов от нейросети.

## Триггеры
- "проверь ответ"
- "прогони через проверщик"
- "проверь это"
- "check this"
- "перепроверь ответ"
- "улучши ответ"

## Что делает
Принимает текст ответа (от Claude или другой нейросети) и прогоняет через 4 фильтра.
Если ответ не проходит хотя бы один — переписывает до приемлемого качества.

## 4 этапа проверки

### Этап 1 — Логика
Проверить:
- Нет ли внутренних противоречий
- Нет ли дыр в рассуждении (пропущенные шаги, необоснованные выводы)
- Выводы следуют из аргументов?

Если есть проблемы → исправить логику, убрать противоречия.

### Этап 2 — Адекватность
Проверить:
- Соответствует ли ответ реальности (нет ли выдуманных фактов, цифр, названий)
- Реалистичны ли рекомендации для малого/среднего бизнеса в России
- Не устарела ли информация

Если есть проблемы → убрать или пометить сомнительные утверждения как «требует проверки».

### Этап 3 — Человечность
Проверить:
- Нет ли канцелярщины («в рамках данного», «осуществить», «произвести»)
- Нет ли ИИ-маркеров («Конечно!», «Отличный вопрос!», «Безусловно»)
- Нет ли шаблонных структур без смысла
- Текст звучит как живой человек?

Если есть проблемы → переписать простым языком, убрать обороты.

### Этап 4 — Соответствие источникам
Проверить:
- Есть ли подтверждение в файлах из `Регламенты/`
- Есть ли подтверждение в материалах из `Хранилище/raw/`
- Если ответ противоречит регламенту — приоритет у регламента

Если подтверждения нет → пометить как «не подтверждено источниками» или убрать.

## Формат результата

```
## Проверка ответа

### Этап 1 — Логика: ✅ / ⚠️ / ❌
[комментарий если есть проблемы]

### Этап 2 — Адекватность: ✅ / ⚠️ / ❌
[комментарий если есть проблемы]

### Этап 3 — Человечность: ✅ / ⚠️ / ❌
[комментарий если есть проблемы]

### Этап 4 — Источники: ✅ / ⚠️ / ❌
[комментарий если есть проблемы]

---
## Итоговый ответ (после правок)
[улучшенная версия или оригинал если всё ок]
```

## Правила
- Если все 4 этапа зелёные — вернуть оригинал без изменений
- Если хотя бы один этап красный или жёлтый — переписать проблемный блок
- Не менять смысл, только качество
- Язык результата — русский
