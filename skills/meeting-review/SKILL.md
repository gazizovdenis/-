# meeting-review

Скилл для разбора транскрипта встречи или звонка.

## Триггеры
- "разбери встречу"
- "разбери звонок"
- "обработай транскрипт"
- "сделай итоги встречи"
- "что решили на встрече"

## Что делает

### Если передан аудиофайл (.m4a, .mp3, .wav, .ogg и др.)
Запустить полный пайплайн:
1. Транскрибировать аудио через скилл `transcribe-media` → получить `_transcript.md`
2. Диаризировать через скилл `diarize-media` → разбить по спикерам
3. Перейти к разбору транскрипта (шаги ниже)

### Если передан готовый транскрипт (.md или текст)
1. Прочитать текст транскрипта
2. Извлечь ключевую информацию по 3 блокам
3. Сформировать готовый текст для Telegram
4. Сохранить результат в `Хранилище/wiki/Все_звонки_и_встречи.md` (дописать в конец)

## Структура результата

### Блок 1 — Краткие итоги
- 3–7 пунктов: что обсудили, что решили, ключевые договорённости
- Без воды, только суть

### Блок 2 — Задачи
Таблица:

| Задача | Ответственный | Дедлайн |
|--------|--------------|---------|

- Если дедлайн не назван — ставить прочерк
- Если ответственный не назван — писать «не назначен»

### Блок 3 — Текст для Telegram
Готовое сообщение для отправки участникам:
- Начинается с даты и названия встречи
- Краткие итоги (3–5 строк)
- Список задач с ответственными
- Без форматирования markdown — только plain text

## Правила
- Не придумывать задачи, которых не было в разговоре
- Если дедлайн упомянут нечётко ("на следующей неделе") — писать как есть
- Имена брать из транскрипта как есть
- Язык результата — русский
