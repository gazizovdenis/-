# transcribe-media

Этот skill нужен для транскрибации аудио и видео файлов в текст.

## Что делает
Когда пользователь просит расшифровать аудио или видео, ты должен:

1. Принять путь к файлу.
2. Проверить, существует ли файл.
3. Проверить наличие инструментов:
   - `ffmpeg`
   - `faster-whisper`
4. Если входной файл — видео, сначала извлечь из него аудио через `ffmpeg`.
5. Сделать транскрибацию через `faster-whisper`.
6. Сохранить результат в markdown-файл.
7. Вернуть пользователю:
   - путь к итоговому файлу
   - короткую сводку, что сделано

## Куда сохранять результат
Если в проекте есть папка `Хранилище`, сохраняй результат в:

`Хранилище/raw/`

Если такой папки нет, создай её.

Имя файла результата:
- взять имя исходного файла
- добавить суффикс `_transcript.md`

Пример:
- вход: `meeting.mp4`
- выход: `Хранилище/raw/meeting_transcript.md`

## Правила работы
- Не изменяй исходный файл.
- Если нет `ffmpeg`, скажи пользователю, что нужен `ffmpeg`.
- Если нет `faster-whisper`, скажи пользователю, что нужен `faster-whisper`.
- Если файл не найден, честно сообщи об этом.
- Если язык не указан, по умолчанию используй русский.
- Если пользователь просит максимум качества, используй более качественную модель.
- Если пользователь просит быстрее, используй более быструю модель.

## Модель по умолчанию
Используй:
- модель: `medium`
- язык: `ru`

Если пользователь пишет:
- "быстро" → можно использовать `small`
- "максимум качества" → можно использовать `large-v3`
- "автоопределение языка" → язык `auto`

## Формат результата
Итоговый markdown-файл должен содержать:

1. Заголовок с именем файла
2. Дату создания
3. Исходный путь
4. Использованную модель
5. Язык
6. Полную расшифровку
7. Если доступны сегменты — добавь таймкоды

Пример структуры:

```md
# Транскрибация: meeting.mp4

- Дата: 2026-05-18
- Исходный файл: /полный/путь/к/meeting.mp4
- Модель: medium
- Язык: ru

## Текст

...

## Таймкоды

[00:00:00] ...
[00:00:12] ...
