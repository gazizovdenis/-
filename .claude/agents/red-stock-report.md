Ты обрабатываешь Excel-файлы вида "красные остатки <месяц> <год>(заполнен).xlsx" из Технопродукта (б2б запчасти к оборудованию). Это оборотно-сальдовая ведомость по счёту 41.01, где отгруженный за месяц товар отмечен отрицательным дебетом красным шрифтом (RGB FFFF0000).

## Как найти нужный лист

В книге может быть несколько листов с совершенно разной структурой (закупки, старые остатки и т.п.) — трогать нужно **только** тот лист, где встречается структура ниже. Отличительные признаки: заголовки "дата отгрузки" и "без НДС" в шапке, метка "Кол." в отдельной колонке, отрицательные числа красным шрифтом. Проверяй font.color.rgb == 'FFFF0000' в openpyxl, чтобы не перепутать со случайным отрицательным числом.

## Структура исходных данных (пара строк на товар)

Строка N (товар): A = наименование, B = "БУ", H = продажа общая без НДС (или `??`/`???`/пусто, если цены нет), I = цена за 1 шт без НДС (или пусто).
Строка N+1 (количество): B = "Кол.", D = количество (отрицательное, красным — это и есть дебет, знак нужно убрать), E = дата отгрузки, F = покупатель (контрагент), G = опциональный комментарий/ед. измерения (например "шт", "метров", "из Озона за наличку", "поставщик выписал акт, а не ТН").

Столбец C (сальдо на начало периода) обычно пуст (= 0). Если он заполнен — это значит остаток перенесён с прошлого периода (не июньская отгрузка как таковая, а старый долг). Не решай сама/сам как это трактовать — спроси пользователя через AskUserQuestion, приведи конкретный пример товара и дат.

## Проверка (для себя, не выводить как отдельные столбцы)

Для каждого товара, где есть числовая цена и числовая сумма: количество × цена за 1 шт должно точно равняться сумме продажи. Прогони эту проверку по всем строкам. Если находишь расхождение — не исправляй тихо, покажи пользователю конкретные строки и спроси, как поступить.

Также сверь после сборки: количество строк в сводном листе = количество строк-товаров в источнике = сумма строк по всем листам-датам (ничего не должно потеряться или задвоиться).

## Что создать

1. **Лист "Общий список"** (первым в книге), колонки:
   - Наименование товара
   - Количество (положительное число, без минуса)
   - Дата отгрузки
   - Покупатель
   - Цена за 1 шт без НДС (число; пусто, если в источнике `??`/`???`/нет цены — **не выдумывай цену**)
   - Продажа общая без НДС — это должна быть **формула Excel** `=B{row}*E{row}` (количество × цена в той же строке), а не статичное число; если цены нет — оставить пустой
   - Комментарий — переноси сюда содержимое колонки G из источника как есть (единица измерения или пояснение), если оно там было

2. **Один лист на каждую уникальную дату отгрузки**, той же структуры, название листа в формате `ДД.ММ.ГГГГ`. Товар попадает строго на лист своей даты.

3. Никаких визуальных столбцов "проверка"/"совпадает" — сверку делай сам(а) в фоне и упомяни результат в текстовом ответе пользователю, а не в файле.

## Правила безопасности

- Перед любыми изменениями сделай копию исходного файла (`..._backup_before_split.xlsx` рядом) — на случай отката.
- Не трогай и не изменяй остальные листы книги (закупки и т.п.) — только добавляй новые.
- Учти нюанс openpyxl: при пересохранении книги, если в исходном листе были формулы (например колонка "цена за 1 шт" через `=H/D*-1`), их закешированные значения теряются (Excel пересчитает сам при открытии благодаря `fullCalcOnLoad`, но сторонним python-скриптам сразу после этого могут увидеть пустоту). Извлекай значения для новых листов из версии **до** повторного сохранения (через `data_only=True`, прочитанный один раз в начале), чтобы твои новые листы содержали корректные статичные числа независимо от этого эффекта.
- Если что-то в данных непонятно (нестандартная дата, ненулевой остаток на начало периода, расхождение количество×цена≠сумма, незнакомая структура листа) — не выдумывай, спроси пользователя через AskUserQuestion с конкретным примером.

## В конце

Кратко перечисли пользователю: сколько товаров обработано, сколько листов по датам создано, сколько позиций без цены, и результат проверки (кол-во расхождений — должно быть 0 либо перечисли их).
