# Шаг 4. Референс — структура Automy AI

Это пример рабочей системы автора курса. К ней нужно стремиться через 1–3 месяца использования. Сейчас вы только закладываете фундамент — не сравнивайте свою недельную сборку с многомесячной системой. Этот файл нужен, чтобы понимать **направление движения**.

---

## Главная идея системы

- **Одна корневая папка** `Система/` на локальном диске. Не Notion, не Drive, не два сервиса одновременно. Один источник истины.
- **Внутри Obsidian Vault** — потому что Obsidian умеет cross-links между файлами через формат `[[имя-файла]]`. ИИ и человек могут переходить между связанными темами одним кликом.
- **Два слоя**: `wiki/` (32 обзорных страницы, читаются первыми) + `raw/` (11 пронумерованных подпапок с оригиналами).
- **Главный файл-проводник** — `CLAUDE.md` в корне. ИИ читает его при каждом запуске.

## Что особенного

### 1. `wiki/index.md` — карта всех страниц

Один файл, через который любой человек или ИИ может попасть на нужный материал. Внутри — две таблицы:

- «Запрос пользователя → стартовая страница» (например, «спрашивают про клинику → открой `klinika-zdorovye.md`»).
- «Все wiki-страницы по группам» (Проекты / Личное / Текущие задачи / Спецтемы).

Это первый файл, который ИИ открывает, когда `CLAUDE.md` не дал прямого маршрута.

### 2. `wiki/_AI_WORKFLOW.md` — playbook с decision tree

Отдельный файл, который описывает алгоритм работы ИИ со всей системой:

- Главный принцип: «все ответы — через систему, не выдумывать».
- Decision tree: ветвление по типу запроса (про проект → wiki/<проект>.md, про человека → ПОРТРЕТ_ДЛЯ_AI.md, и т.д.).
- Cheatsheet топ-10 типовых запросов с цепочкой файлов.
- Workflow обновления данных: «когда новый факт → положить в Х → сообщить кратко».

Этот файл — продвинутая опция. Появляется на 2–3 месяце, когда понимаешь, какие типовые запросы повторяются.

### 3. Cross-links через `[[wiki-link]]`

Все wiki-страницы ссылаются друг на друга в формате Obsidian: `[[klinika-zdorovye]]`. Это позволяет:

- ИИ переходить между связанными темами без поиска по тексту.
- В Obsidian видеть граф связей всех файлов визуально.
- При переименовании файла все ссылки на него обновляются автоматически.

### 4. Блок `Sources:` в каждой wiki-странице

Каждая обзорная страница в начале указывает, из каких файлов в `raw/` она собрана:

```markdown
**Sources:**
- raw/02_Клиника_Здоровье/README.md
- raw/02_Клиника_Здоровье/03_Встречи/ЗАДАЧИ_СО_ЗВОНКА_2026-02-02.txt
- raw/02_Клиника_Здоровье/01_Маркетинг/
```

Это «доказательство»: если ИИ или человек сомневается в цифре или факте — открывает источник и сверяется. Без этого блока wiki быстро превращается в выдумку.

### 5. `Заметки_чат.md` в каждом проекте

В каждой проектной подпапке `raw/0X_<проект>/` лежит `Заметки_чат.md` — хронологический журнал решений, договорённостей, операционных деталей. Свежие записи сверху.

Зачем: ИИ читает этот файл, когда нужно понять «что происходило по проекту в последние недели». Без журнала пришлось бы перебирать все транскрипты подряд.

### 6. `README.md` в каждой подпапке `raw/`

В **каждой** проектной и сферной подпапке — свой `README.md`. Первая строка — напоминание «Перед задачей — прочитай <АБСОЛЮТНЫЙ_ПУТЬ>/CLAUDE.md». Дальше — описание структуры подпапки и таблица «куда что класть».

Это критически важно: ИИ не всегда сам открывает корневой `CLAUDE.md` при работе из подпапки. README с напоминанием — единственный способ это починить.

### 7. Иерархия источников истины

В `CLAUDE.md` явно прописана последовательность, кому верить при конфликте:

1. `raw/08_Личное/ПОРТРЕТ_ДЛЯ_AI.md` — стиль общения, табу, приоритеты.
2. `raw/08_Личное/ГЛАВНЫЙ_ФАЙЛ_О_СЕБЕ.md` — факты о владельце.
3. `CLAUDE.md` — рабочие правила.
4. `wiki/<проект>.md` — обзоры проектов.
5. `raw/0X_<проект>/README.md` — структура папки.
6. `raw/0X_<проект>/Заметки_чат.md` — рабочий журнал (самое свежее).

И отдельно: при конфликте `wiki` и `raw` — всегда верим `raw`, потом синхронизируем `wiki`.

### 8. Журнал переноса в Мусорке

`raw/00_Мусорка/_log.md` — таблица всех переносов файлов: дата / источник / цель / причина / кто перенёс. Никаких `rm`, только `mv` с записью в журнал. Это даёт возможность восстановить любой «удалённый» файл через год.

## К чему стремиться (числовые ориентиры)

| Метрика | Стартовая система (после Модуля 01) | Зрелая система (Automy AI) |
|---------|-------------------------------------|----------------------------|
| Wiki-страниц | 3–5 (Портрет + index + 1–3 проекта) | 30+ |
| Проектных подпапок в `raw/` | 1–3 | 6–11 |
| Файлов в `raw/` | 20+ | 1000+ |
| README в подпапках | в каждой проектной | в **каждой** подпапке raw |
| Decision tree (`_AI_WORKFLOW.md`) | нет | есть |
| Cross-links между wiki | нет | плотные |
| Журнал в Мусорке | пустой `_log.md` | заполняется при каждом переносе |

Не пытайтесь догнать это за неделю. Реалистичный темп: 5–10 wiki-страниц за первый месяц, остальное приходит само по мере использования.
