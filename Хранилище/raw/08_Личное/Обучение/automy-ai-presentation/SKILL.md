# automy ai · Presentation

Генерирует слайдовую презентацию в фирменном стиле курса automy ai. Эталон — `Obsidian Vault/Хранилище/raw/07_AI/Курс/Модуль 02/Презентация_Модуль_02.html`.

На выходе: папка рядом с исходником, в ней `<имя>.html` (интерактивный 16:9 дек), `<имя>.pdf` (рендер всех слайдов) и два логотипа automy ai.

---

## Когда вызывать

Никита просит:
- «сделай презентацию automy ai / для курса / для модуля X»
- «слайды в стиле модуля 02»
- «оформи как презентацию automy»
- «свёрстай модуль / урок / лекцию» (если упоминается курс по AI/Cursor)

Если просит «презентацию» без указания стиля — уточни через AskUserQuestion: «automy ai курс» или другой стиль.

## Чего этот skill НЕ делает

- Не делает PowerPoint / Keynote / Google Slides — только self-contained HTML + Chrome→PDF.
- Не подключает фреймворков (Tailwind/React) — всё inline в `<style>`.
- Не генерирует видеосодержание, не пишет сценарий — только верстает по структуре, которую дал Никита (или которую достал из исходного MD).

## Алгоритм работы

### 1. Получить вход
- Если Никита дал MD-файл / план / маркдаун — взять оттуда заголовки секций → слайды.
- Если дал свободный текст — разбить на 8–20 слайдов (титул → тезис → раскрытие → правило → CTA/следующий шаг).
- Если непонятно — спросить через AskUserQuestion одним вопросом: тема модуля + 3–5 пунктов содержания.

### 2. Подготовить папку
Папка-приёмник: рядом с исходным файлом, имя = базовое имя без расширения.

```
<родитель>/
  └── <имя>/
      ├── <имя>.html
      ├── <имя>.pdf
      ├── automy-ai-wordmark-display-64-orange.png
      └── automy-ai-logo-light-on-dark.png
```

Шаги:
- Создать папку. Если уже есть — НЕ перезаписывать содержимое слепо. Если внутри есть свежий `<имя>.html` (повторный запуск этим же скилом) — перезаписывать молча. Если внутри что-то постороннее — спросить.
- Скопировать оба логотипа из `~/.claude/skills/automy-ai-presentation/assets/` в папку через `cp`.
- Если был исходный MD — перенести его в эту же папку через `mv` (не копировать).

### 3. Сгенерировать HTML
- Базовый каркас брать из `~/.claude/skills/automy-ai-presentation/template.html`.
- Каркас содержит: `<style>` с дизайн-системой, `<div class="stage"><div class="deck">` с примерами 7 типовых слайдов, навигацию `.nav`, JS-обработчик стрелок/клавиш, авто-нумерацию слайдов в `.head .num`.
- Заменить слайды-примеры на реальные слайды по содержанию. Структура каждого слайда — из «Библиотеки компонентов» ниже.
- Не трогать `<style>` и `<script>` — это фундамент стиля.

### 4. Сгенерировать PDF
- Запустить `~/.claude/skills/automy-ai-presentation/html_to_pdf.sh "<абсолютный путь к html>"`.
- Скрипт сам положит PDF рядом с HTML (заменит `.html` на `.pdf`).
- Если Chrome не найден — скрипт вернёт ошибку с инструкцией ручного экспорта; сообщить Никите и предложить открыть HTML и нажать Cmd+P.

### 5. Сообщить результат
- Markdown-ссылку на папку
- Markdown-ссылку на HTML и на PDF
- Команду `open "<абсолютный путь к html>"` для просмотра в браузере
- 1 строкой: число слайдов, размер PDF

## Дизайн-система (зафиксирована, не отклоняться)

### Палитра

| Переменная | HEX | Где применяется |
|-----------|------|-----------------|
| `--orange` | `#ED7E2F` | Главный акцент: заголовки-выделения, цифры, accent-карточки, рамки, ссылки в коде |
| `--orange-soft` | `#FFE6D1` | Soft-фоны карточек (вторичный акцент) |
| `--orange-pale` | `#FFF4E8` | Самые лёгкие фоны (редко) |
| `--ink` | `#0F0F0F` | Основной текст и тёмные слайды |
| `--ink2` | `#6B6B6B` | Subtitle, описания |
| `--bg` | `#FFFFFF` | Фон светлых слайдов |
| `--gray` | `#F2F0EC` | Бежевый фон второстепенных карточек |
| `--line` | `#E5E3DE` | Тонкие разделители и рамки |
| `--dark` | `#0F0F0F` | Тёмные слайды и tree-блоки |

Контраст: на белом — `--ink` + `--orange`. На `--dark` — белый + `--orange`. Никаких других ярких цветов.

### Шрифты
- **Inter** — весь UI, заголовки, текст. Веса: 400/500/600/700/800/900.
- **JetBrains Mono** — код, имена файлов, нумерация слайдов, технические термины.
- Никаких других шрифтов.

### Геометрия
- Слайд: 16:9. Внутренние отступы `44px 60px 60px`.
- Радиусы: карточки `16–22px`, pill `999px`, code `14px`.
- @page в print: `13.333in × 7.5in` (стандарт 16:9 PDF).

### Заголовки
- `h1` — 80px, weight 900, letter-spacing -.025em
- `h1.giant` — 104px
- `h1.mega` — 120px (только для слайдов-разделителей)
- `h2` — 58px, weight 900
- `h3` — 28px, weight 800

### Что нельзя
- ❌ Другие цвета (синий, зелёный, фиолетовый) — только палитра выше.
- ❌ Тяжёлые тени, градиенты, glassmorphism.
- ❌ Иконочные шрифты (FontAwesome и т.п.). Можно — текстовые эмодзи (💻🖥📄✈⌥).
- ❌ Скруглять слайд изнутри — рамка слайда уже скруглена через `.deck`.
- ❌ Менять `--orange` на другой оттенок.
- ❌ Перегружать слайд: максимум 1 заголовок + 1 подзаголовок + 1 блок-компонент. Если контента больше — разбей на 2 слайда.

## Архитектура слайда

Каждый `<section class="slide">` (или `slide dark`) состоит из:

```html
<section class="slide">           <!-- или "slide dark" для тёмного -->
  <div class="head">              <!-- шапка с логотипом и нумерацией -->
    <div class="brand">
      <img class="logo-mark light" src="automy-ai-wordmark-display-64-orange.png" alt="automy ai">
      <img class="logo-mark dark"  src="automy-ai-logo-light-on-dark.png"        alt="automy ai">
      <span class="section-label">ЯРЛЫК СЕКЦИИ</span>
    </div>
    <span class="num"></span>     <!-- авто-заполняется JS: "01 / 20" -->
  </div>
  <hr class="divider">
  <div class="body">              <!-- основной контент -->
    <h2>Заголовок слайда</h2>
    <div class="sub">Подзаголовок одной строкой</div>
    ...компонент из библиотеки ниже...
  </div>
</section>
```

**`section-label`** — короткий ярлык (1–2 слова, строчными), отражающий тип слайда: `модуль 02`, `тезис`, `схема`, `сравнение`, `команды`, `зачем`, `инструмент`, `установка`, `фича`, `подход`, `магия`, `правило`, `поток`, `шаги`, `чеклист`, `следующий модуль`.

**Логотипы** оба добавляются всегда — CSS сам показывает нужный в зависимости от `.slide.dark`.

## Библиотека компонентов (что выбирать под задачу)

### `.title-row` — титульный слайд
Левая часть: гигантский заголовок + сабтайтл. Правая: чёрная `.module-card` с большой цифрой модуля.

```html
<div class="body title-row">
  <div class="title-left">
    <div>
      <h1 class="giant">Название<br>в две-три<br>строки</h1>
      <div class="sub" style="margin-top:22px;font-size:22px">Подзаголовок что внутри модуля</div>
    </div>
  </div>
  <div class="module-card">
    <div><div class="top">МОДУЛЬ</div><div class="dot"></div></div>
    <div class="num">02</div>
    <div class="tag"><b>Короткое имя</b><span>тег внизу</span></div>
  </div>
</div>
```

### `.quote-wrap` — слайд-тезис (обычно `.slide.dark`)
Огромный заголовок + одна строка-расшифровка. Применять для разделителей-манифестов.

```html
<div class="body quote-wrap">
  <h1 style="color:#fff;font-size:84px;font-weight:900;line-height:1;letter-spacing:-.025em">
    Файлы — это<br>ещё не знание.<br>Знание — <span class="accent">связи</span>
  </h1>
  <div class="quote-sub lime" style="margin-top:30px;font-size:21px">Подзаголовок-пояснение</div>
</div>
```

### `.bricks` — 3 горизонтальные «кирпичные» полоски с номерами
Идеально для «3 пункта / 3 правила / 3 шага». Третий — `.brick.accent` (оранжевый).

```html
<div class="bricks" style="gap:14px;margin-top:24px">
  <div class="brick" style="padding:22px 32px"><div class="n">01</div><div><div class="ttl">Первый</div><div class="desc">Описание</div></div></div>
  <div class="brick" style="padding:22px 32px"><div class="n">02</div><div><div class="ttl">Второй</div><div class="desc">Описание</div></div></div>
  <div class="brick accent" style="padding:22px 32px"><div class="n">03</div><div><div class="ttl">Третий</div><div class="desc">Главный — оранжевый</div></div></div>
</div>
```

Модификаторы: `.brick.soft` (бежевый), `.brick.gray`, `.brick.dark` (для тёмного слайда).

### `.grid4` — 4 карточки в ряд (с большой цифрой)
Для «4 шага установки», «4 метрики», «4 факта». Последняя — `.card.accent`.

```html
<div class="grid4">
  <div class="card"><div class="k">01</div><div class="t">Заголовок</div><div class="d">Описание</div></div>
  <div class="card"><div class="k">02</div><div class="t">Заголовок</div><div class="d">Описание</div></div>
  <div class="card"><div class="k">03</div><div class="t">Заголовок</div><div class="d">Описание</div></div>
  <div class="card accent"><div class="k">04</div><div class="t">Итог</div><div class="d">Финал</div></div>
</div>
```

### `.col3` — 3 большие карточки
Для «3 инструмента / 3 команды / 3 уровня». Можно одной поставить `.accent` или `.soft`.

```html
<div class="col3">
  <div class="card" style="padding:20px 22px">
    <h3>Вариант 1</h3>
    <div style="font-size:15px;line-height:1.5">Описание</div>
  </div>
  <div class="card">...</div>
  <div class="card accent">...</div>
</div>
```

### `.col2` — 2 колонки с заголовками
Для сравнений «А vs Б». Левая — нейтральная, правая — `.card.dark` или `.card.accent`.

```html
<div class="col2">
  <div class="card">
    <div class="lbl">ЛЕЙБЛ ВВЕРХУ</div>
    <h3>Заголовок</h3>
    <ul><li>пункт</li><li>пункт</li></ul>
  </div>
  <div class="card dark">
    <div class="lbl">НАШ ПОДХОД</div>
    <h3>Заголовок</h3>
    <ul><li>пункт</li><li>пункт</li></ul>
  </div>
</div>
```

### `.grid2x2` — 4 компактные карточки 2×2
Когда нужна сетка 2×2. Можно одну растянуть на всю ширину: `.full.accent`.

### `.steps2col` — чеклист на 6 шагов
6 нумерованных полосок в 2 колонки × 3 строки. Последний — `.start` (оранжевый).

```html
<div class="steps2col">
  <div class="step"><div class="num">01</div><div><div class="t">Шаг</div><div class="d">Действие</div></div></div>
  ...
  <div class="step start"><div class="num">06</div><div><div class="t">Финал</div><div class="d">...</div></div></div>
</div>
```

### `.tree` — структура папки / схема (тёмный код-блок)
```html
<div class="tree">
<pre>
Система/
├── <span class="or">raw/</span><span class="g">сырые файлы</span>
├── <span class="or">wiki/</span><span class="g">обзоры со ссылками</span>
└── CLAUDE.md
</pre>
</div>
```

### `.code` — inline-код или код-блок
```html
<div class="code">$ <span class="or">ingest new data</span></div>
<div class="code">Встретился с <span class="or">[[Дизайнером]]</span> по <span class="or">[[Проекту A]]</span></div>
```

### `.flow` — поток нод со стрелками
```html
<div class="flow">
  <div class="node">Входные данные</div>
  <div class="arrow">↓</div>
  <div class="node accent">Обработка</div>
  <div class="arrow">↓</div>
  <div class="node dark">Результат</div>
</div>
```

### `.pills` — теги-таблетки
```html
<div class="pills">
  <div class="pill">тег</div>
  <div class="pill accent">главный тег</div>
</div>
```

### `.foot-grid` — футер с куратором (на финальных слайдах)
```html
<div class="foot-grid" style="grid-template-columns:1fr">
  <div>
    <div class="lbl">куратор</div>
    <div class="at">@Anti4ay</div>
    <div class="desc">Никита</div>
  </div>
</div>
```

### `.center` — центрирование контента (для слайдов-правил)
```html
<div class="body center">
  <h1 style="color:#fff;font-size:80px;text-align:center">ГЛАВНОЕ <span class="accent">ПРАВИЛО</span></h1>
  <div class="quote-sub" style="text-align:center;margin-top:34px">Подпись по центру</div>
</div>
```

### SVG-схемы
Для нетривиальных иллюстраций (граф, архитектура, сравнения процессов) — используй inline `<svg viewBox="0 0 1180 410">`. Правила:
- Заливки только из палитры (`#ED7E2F`, `#0F0F0F`, `#fff`, `#f4f1ec`, `#d8d6d0`).
- Стрелки — через `<marker>` с `fill="#0F0F0F"` или `#ED7E2F`.
- Текст — `font-family="Inter, sans-serif"` или `"JetBrains Mono, monospace"`, font-weight 700–800.
- Подписи разделов в SVG — uppercase, letter-spacing 1.5–2, цвет `#888`.

Образцы рабочих SVG — в эталоне `Презентация_Модуль_02.html` (слайды 3, 4, 7, 9, поток, следующий модуль).

## Структура хорошей презентации (10–18 слайдов)

Типовая последовательность:

1. **Титул** — `.title-row` + `.module-card`
2. **Тезис** — `.slide.dark` + `.quote-wrap` (главная идея модуля)
3. **Схема** — SVG-иллюстрация что устроено как
4. **Сравнение / противопоставление** — `.col2` (наш подход vs обычный)
5. **3 ключевых пункта** — `.bricks` или `.col3`
6. **Зачем / польза** — `.bricks` с 3 пунктами
7. **Инструмент** — left/right grid (описание + скриншот SVG)
8. **Установка / шаги** — `.grid4` (4 шага)
9. **Главная фича** — `.slide.dark` с большим JetBrains Mono элементом
10. **Подход** — `.bricks`
11. **Граф / визуализация** — SVG
12. **Правило** — `.slide.dark` + `.center` (один заголовок капсом)
13. **Поток данных** — SVG sankey
14. **Преимущества** — `.col3`
15. **Финал / чеклист** — `.steps2col` (6 шагов)
16. **Следующий модуль** — `.slide.dark` с превью + `.foot-grid` с куратором

Не обязательно все — но соблюдай ритм: чёрный/белый чередуй (черный слайд через 2–4 белых).

## Текстовые правила (тон голосом Никиты)

- **Заголовок** — 2–6 слов. Если длиннее — разбей на 2 строки `<br>`.
- **Подзаголовок** — 1 предложение, до 20 слов.
- **Описание в карточке** — 1–2 коротких предложения, без воды.
- **Тон** — прямой, без «как известно», «следует отметить». Глагол в начале: «Кидаете в raw/», «Получаете граф».
- **Цифры** — всегда с ведущим нулём («01», «02»), JetBrains Mono.
- **Termin'ы файлов / папок** — моноширинно: `<span style="font-family:'JetBrains Mono', monospace">raw/</span>`.
- **Wiki-ссылки** — `[[имя]]` в оранжевом цвете внутри `.code` или прямо в тексте.
- **CTA в конце** — `→ короткий вывод одной строкой` через `.small`.
