# Гульнара — памятка по закупу и логистике (для передачи в работу)

Адаптация регламента для практического использования. Полный текст —
`Хранилище/raw/Регламенты/Регламент_закуп_и_логистика_финал.docx`, при спорных
случаях сверяться с ним.

## Зона ответственности (коротко)

Ты ведёшь поставку от заявки до финального получения товара:
- проверяешь и готовишь файлы закупа;
- считаешь суммы и запускаешь оплаты через ответственного за переводы;
- контролируешь сроки, статусы, накладные и логистические расходы;
- фиксируешь всё в рабочих таблицах;
- сверяешь план/факт по отгрузке и приходу;
- координируешь движение товара Китай → Москва → Уфа/фулфилмент;
- сообщаешь менеджерам статусы;
- по действующим поставщикам ставишь задачи Лене.

**Не входит:** переговоры с ключевыми поставщиками, выбор поставщика,
согласование новой цены/объёма/замены товара, спорные и рисковые решения —
это Иван. Твоя задача в таких случаях — не решать самой, а передать вопрос
(см. «Когда эскалировать» ниже).

## Процесс по новинкам (Лена не участвует)

Связка: менеджер → ты → Игорь, с подключением Ивана при необходимости.

1. **Приём заказа** — менеджер присылает заполненный файл «Заказ в Карго».
   Проверяешь на полноту и корректность. Ошибки — возвращаешь менеджеру на
   доработку. Всё ок — отправляешь Игорю.
2. **Проверка наличия** — Игорь связывается с поставщиками, сообщает что есть
   в наличии, чего нет, что требует замены. Передаёшь это менеджерам, они
   вносят правки, ты проверяешь финальную версию и подтверждаешь готовность
   к закупу.
3. **Оплата закупа** — Игорь считает итоговую сумму в файле «Заказ в Карго».
   Ты отправляешь запрос на перевод ответственному за переводы, после оплаты
   получаешь чек и пересылаешь его Игорю. Игорь запускает закуп.
4. **Контроль закупа** — если Игорь сам не написал статус, запрашиваешь его
   через **2–3 дня после оплаты** (или в срок, отдельно оговорённый по
   заказу). После готовности товара уточняешь у менеджеров способ доставки,
   срочность и нужна ли доп. упаковка.
5. **Накладная** — после отправки Игорь обязан прислать накладную и стоимость
   доставки. Не пришёл — запрашиваешь повторно. Вносишь данные в файл
   «Мой склад задачи»: товар, накладная, стоимость, дата отправки, способ
   доставки, комментарии.
6. **Москва → дальше** — по прибытии в Москву получаешь номер накладной,
   сумму к оплате и параметры груза → запрос на оплату Китай–Москва →
   после оплаты подтверждение логисту → передаёшь данные для дальнейшей
   отправки (ТК, склад назначения, получатель, телефон). По габаритным/
   тяжёлым грузам по умолчанию — склад Айваза, если не оговорено иначе.
   Расходы — в «Мой склад задачи».

## Процесс по действующим поставщикам (через Лену)

1. Готовишь файлы закупа.
2. Передаёшь их Лене — она пишет поставщику в WeChat: наличие, цены, фото/
   видео, сроки производства и поставки, подтверждение отправки.
3. Всё, что узнала Лена, сразу идёт тебе.
4. Ты отражаешь в таблицах, ведёшь оплаты, статусы, накладные, логистику,
   план/факт — как и по новинкам.

Лена не выбирает поставщика, не согласовывает цену/объём/замену/отсрочку —
любой такой вопрос от неё идёт тебе, а если нестандартный — ты подключаешь
Ивана.

## Что и где фиксировать

По каждой поставке в **«Мой склад задачи»**: товар, поставщик, номер
накладной, стоимость закупа, стоимость логистики, вес/объём, тип доставки
(быстрая/обычная/срочная), даты этапов, склад назначения, комментарии,
итоговый статус.

По каждой поставке отслеживаются даты: оплата закупа → готовность товара →
отправка в карго → отправка карго в Москву → прибытие в Москву → оплата
логистики Китай–Москва → отправка в Уфу/на фулфилмент → финальное получение.

**Правило:** вносить данные не позже дня, в который получена информация. Не
держать статусы только в переписке.

## Когда решаешь сама, когда зовёшь Ивана

**Сама:** запрос/повторный запрос статусов, сбор документов, напоминания о
накладных, фиксация в таблицах, стандартная передача данных по ТК/складу/
получателю, информирование менеджеров, типовая логистика, отклонения по
срокам в пределах нормы.

**Обязательно к Ивану:** не хватает части ключевого товара; нужна
существенная замена; поставщик предлагает новые условия; срок поставки
существенно сдвигается; нужна нестандартная упаковка с доп. затратами; спор
по составу заказа; риск срыва запуска продаж; нестандартный платёж или
логистическая схема.

## Еженедельный отчёт по поставкам

Отдельно от твоей финансовой отчётности — по каждой активной поставке:
товар, поставщик, стадия, плановая дата, фактическая дата, отклонение, сумма
закупа, сумма логистики, текущий риск, действия, нужно ли решение Ивана.

## Запрещено

Оставлять заказ без контрольной точки; не фиксировать накладную; не вносить
расходы в таблицу; держать статусы только в переписке; перекладывать
стандартные действия на Ивана; согласовывать критические изменения без
эскалации.
