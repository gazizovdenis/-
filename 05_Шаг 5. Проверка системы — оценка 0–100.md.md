# Шаг 5. Проверка системы — оценка 0–100

Шаги 1–3 пройдены. Сейчас запустим автоматический аудит: ИИ просканирует вашу папку `Система/`, оценит её по 6 рубрикам и выдаст итоговый балл от 0 до 100+ плюс список приоритетных правок.

---

## Как работает оценка

Эталон сравнения — рабочая система автора курса (см. файл `04_Структура_Automy_AI.md`).

| Балл | Что значит |
|------|-----------|
| 95–100 | Система сопоставима с эталоном. Готова к ежедневной работе. |
| 100+ | Превзошли эталон (больше wiki, плотнее cross-links, или есть нестандартные дополнения). |
| 70–94 | Хорошая основа, есть точки роста. ИИ уже работает с контекстом, но не на полную мощность. |
| 40–69 | Базовая структура есть, но мало наполнения или нет wiki. Возвращайтесь к шагу 2. |
| 0–39 | Только CLAUDE.md или ничего. Возвращайтесь к шагу 1. |

Оценка делится на 6 рубрик с весами:

| Рубрика | Вес |
|---------|-----|
| CLAUDE.md в корне | 20 |
| Наполнение Хранилища | 20 |
| Разделение raw / wiki | 15 |
| README и Sources | 15 |
| Навигация / index | 15 |
| Связь файлов (cross-links, журнал, decision tree) | 15 |
| **Итого** | **100** |

Бонусные баллы (+1 до +10) — за нестандартные дополнения: decision tree, _AI_WORKFLOW.md, продвинутый журнал в Мусорке, плотные cross-links через `[[wiki-link]]`.

---

## Как запустить

1. Откройте Claude Code в корне вашей папки `Система/`.
2. Скопируйте всё содержимое блока `START PROMPT` … `END PROMPT` ниже.
3. Вставьте в Claude Code как обычное сообщение.
4. ИИ просканирует папку (использует Bash / Read / Glob), посчитает метрики и выдаст отчёт.

Времени уйдёт 3–5 минут на сканирование + чтение отчёта.

---

```text
START PROMPT
============

Ты — аудитор системы контекста для ИИ. Я открыл Claude Code в корне моей папки Система/. Просканируй её структуру и оцени по 6 рубрикам.

ЗАДАЧА: посчитать конкретные метрики (не «на глаз»), выставить балл 0–100+ и дать топ-5 приоритетных правок.

РАЗРЕШЁННЫЕ ИНСТРУМЕНТЫ: Bash (ls, find, wc), Glob, Read. Файлы НЕ изменяй — только сканируй.

═══════════════════════════════════════════════════════
АЛГОРИТМ СКАНИРОВАНИЯ
═══════════════════════════════════════════════════════

ШАГ 1. Найди CLAUDE.md в корне.
   - Команда: `ls -la CLAUDE.md` или `find . -maxdepth 2 -name 'CLAUDE.md'`.
   - Если нет — оценка по рубрике 1 = 0.

ШАГ 2. Если CLAUDE.md есть — прочитай его и проверь:
   - Длина файла (wc -l) — менее 50 строк = неполный.
   - Есть ли заголовки: «Владелец», «Структура», «Правила», «Проекты», «Навигация».
   - Заполнены ли блоки реальными данными или висят плейсхолдеры `[ЗАПОЛНИТЬ:`.
   - Есть ли таблица «тип запроса → стартовый файл».

ШАГ 3. Просканируй Хранилище/:
   - Сколько подпапок верхнего уровня (`find Хранилище -maxdepth 1 -type d`).
   - Сколько .md файлов всего (`find Хранилище -name '*.md' | wc -l`).
   - Сколько проектных подпапок (с префиксом `0X_`).
   - Есть ли `08_Личное/`, `95_Текущие_задачи/`, `00_Мусорка/`.

ШАГ 4. Проверь разделение raw / wiki:
   - Есть ли `Хранилище/raw/` и `Хранилище/wiki/`.
   - Сколько файлов в wiki/.
   - Есть ли `wiki/Портрет.md`, `wiki/index.md`.

ШАГ 5. Проверь README и Sources:
   - В скольких проектных подпапках есть README.md (`find Хранилище -name 'README.md' | wc -l`).
   - В скольких wiki-страницах есть блок `Sources:` (grep по wiki).
   - В скольких README первая строка содержит напоминание «прочитай CLAUDE.md» (grep).

ШАГ 6. Проверь навигацию:
   - Есть ли таблица «запрос → файл» в `wiki/index.md` или в CLAUDE.md.
   - Есть ли отдельный decision tree файл (например, `wiki/_AI_WORKFLOW.md`).

ШАГ 7. Проверь связь файлов:
   - Сколько `[[wiki-link]]` ссылок во всех файлах wiki (grep '\[\[' wiki/*.md | wc -l).
   - Есть ли журнал в Мусорке (`raw/00_Мусорка/_log.md`).
   - В скольких файлах в `raw/` есть `Заметки_чат.md` (find).

═══════════════════════════════════════════════════════
РУБРИКИ И ШКАЛЫ ОЦЕНОК
═══════════════════════════════════════════════════════

РУБРИКА 1. CLAUDE.md (20 баллов)

- 0 баллов: файла нет.
- 5 баллов: файл есть, но <30 строк или почти все плейсхолдеры.
- 10 баллов: файл есть, базовые блоки заполнены (Владелец + 1 проект), но нет Навигации.
- 15 баллов: 5 блоков заполнены реальными данными, есть Навигация без таблицы.
- 20 баллов: всё заполнено + таблица Навигации + блок «Удаление файлов» + ограничения и приоритеты.

РУБРИКА 2. Наполнение Хранилища (20 баллов)

Считай уникальные .md файлы в Хранилище/, исключая README.md и Заметки_чат.md (служебные).

- 0 баллов: Хранилища нет или меньше 5 файлов.
- 5 баллов: 5–19 файлов.
- 10 баллов: 20–49 файлов + 1–2 проектных подпапки.
- 15 баллов: 50–199 файлов + 3+ проектных подпапки + есть 08_Личное и 95_Текущие_задачи.
- 20 баллов: 200+ файлов + 5+ проектных подпапок + полная структура (Личное, Текущие задачи, Мусорка, Остальное).

РУБРИКА 3. Разделение raw / wiki (15 баллов)

- 0 баллов: всё лежит плоско в Хранилище, raw/ и wiki/ нет.
- 5 баллов: есть raw/ но wiki/ нет (или пустая).
- 10 баллов: есть и raw/ и wiki/, в wiki/ 1–4 файла, есть Портрет.md или index.md.
- 15 баллов: есть оба, в wiki/ 5+ файлов, есть и Портрет.md, и index.md, и по странице на каждый проект.

РУБРИКА 4. README и Sources (15 баллов)

- 0 баллов: README нет нигде, кроме корня.
- 5 баллов: README только в части проектных подпапок (<50%).
- 10 баллов: README в каждой проектной подпапке, но без напоминания «прочитай CLAUDE.md» в первой строке.
- 15 баллов: README в **каждой** проектной подпапке (и большинстве вложенных), первая строка — напоминание про CLAUDE.md, в wiki-страницах блок `Sources:` присутствует везде.

РУБРИКА 5. Навигация / index (15 баллов)

- 0 баллов: таблицы «запрос → файл» нигде нет.
- 5 баллов: есть только в CLAUDE.md, нет wiki/index.md.
- 10 баллов: есть wiki/index.md с таблицей, но без группировки по темам.
- 15 баллов: есть wiki/index.md с двумя таблицами (по типам запросов + по группам страниц), есть decision tree (`wiki/_AI_WORKFLOW.md` или эквивалент).

РУБРИКА 6. Связь файлов (15 баллов)

- 0 баллов: cross-links нет, журнал Мусорки пустой, Заметки_чат.md нигде.
- 5 баллов: есть какие-то ссылки или Заметки_чат, но единично.
- 10 баллов: `Заметки_чат.md` есть в каждой проектной подпапке, в Мусорке есть `_log.md` с записями.
- 15 баллов: cross-links через `[[wiki-link]]` плотные (>20 ссылок в wiki), `Заметки_чат.md` в каждом проекте, журнал Мусорки ведётся, явно прописана иерархия источников истины в CLAUDE.md.

БОНУС (+1 до +10)
- +2 за каждое: отдельный `_AI_WORKFLOW.md`, прописанные правила обновления данных, шаблоны для типовых документов, явные cheatsheet для AI, структура `agents/` или `skills/`.

═══════════════════════════════════════════════════════
ФОРМАТ ОТЧЁТА
═══════════════════════════════════════════════════════

После сканирования выдай отчёт строго в этом формате:

──────────────────────────────────────────
АУДИТ СИСТЕМЫ — <дата YYYY-MM-DD>
──────────────────────────────────────────

ИТОГО: XX / 100 (+ Y бонусных) = ZZ баллов

Вердикт: <одна строка — «готова к работе» / «нужно доделать наполнение» / «вернитесь к шагу N»>

──────────────────────────────────────────
МЕТРИКИ ПО РУБРИКАМ
──────────────────────────────────────────

1. CLAUDE.md: XX / 20
   - Файл: есть / нет
   - Длина: NN строк
   - Заполненных блоков: M из 5
   - Плейсхолдеров `[ЗАПОЛНИТЬ:`: K
   - Таблица «Навигация»: есть / нет
   Комментарий: <1–2 строки>

2. Наполнение Хранилища: XX / 20
   - Подпапок верхнего уровня: N
   - .md файлов всего: K
   - Проектных подпапок (0X_): M
   - Есть 08_Личное: да / нет
   - Есть 95_Текущие_задачи: да / нет
   - Есть 00_Мусорка: да / нет
   Комментарий: <1–2 строки>

3. Разделение raw / wiki: XX / 15
   - Есть raw/: да / нет
   - Есть wiki/: да / нет
   - Файлов в wiki/: N
   - Есть Портрет.md: да / нет
   - Есть index.md: да / нет
   Комментарий: <1–2 строки>

4. README и Sources: XX / 15
   - README в проектных подпапках: M из N
   - README с напоминанием про CLAUDE.md: K из N
   - Wiki-страниц с блоком Sources: M из N
   Комментарий: <1–2 строки>

5. Навигация / index: XX / 15
   - wiki/index.md: есть / нет
   - Таблиц в index: 0 / 1 / 2
   - Decision tree (_AI_WORKFLOW): есть / нет
   Комментарий: <1–2 строки>

6. Связь файлов: XX / 15
   - [[wiki-link]] ссылок в wiki/: N
   - Заметки_чат.md в проектах: M из N
   - Журнал Мусорки (_log.md): есть / нет / ведётся
   - Иерархия источников истины в CLAUDE.md: есть / нет
   Комментарий: <1–2 строки>

БОНУС: +X (за <что именно>)

──────────────────────────────────────────

──────────────────────────────────────────

1. <Самая важная — что добавить или исправить, чтобы поднять балл максимально>
2. <Вторая по важности>
3. <Третья>
4. <Четвёртая>
5. <Пятая>

Для каждой правки: одна строка «что сделать» + одна строка «сколько баллов это даст».

──────────────────────────────────────────
СРАВНЕНИЕ С ЭТАЛОНОМ
──────────────────────────────────────────

Эталон (Automy AI) = ~95–100 баллов. Ваша система = ZZ.

<Одно предложение — на каком этапе развития вы находитесь, чего не хватает чтобы дойти до эталона>

──────────────────────────────────────────

ВАЖНО:

- Считай реальные цифры через Bash, не «на глаз».
- Если какой-то файл / подпапка по нумерации отсутствует — это не штраф, просто не считай (например, проектов может быть 1, а не 6 — норма).
- Бонусные баллы добавляй только за то, что реально нашёл, не «на потенциал».
- Топ-5 правок — по убыванию ценности. Первая = максимальный эффект на балл.

Запускай сканирование. Жди отчёт от меня — нужно ли подробнее объяснить какую-то правку.

============
```

---

## Как читать оценку

| Балл | Что делать |
|------|-----------|
| 90+ | Система готова к работе. Используйте её 2–4 недели, потом запустите проверку снова. |
| 70–89 | Возьмите топ-2 правки из отчёта и доделайте за вечер. Через 3 дня запустите проверку повторно. |
| 40–69 | Возвращайтесь к файлу `02_Наполнение_Хранилища.md`. Не хватает материалов или README. |
| 0–39 | Возвращайтесь к файлу `01_Настройка_CLAUDE_MD.md`. Базы пока нет. |

---

## Что делать с топ-5 правками

ИИ выдаст список конкретных правок (например: «создай wiki/Портрет.md», «добавь README.md в Хранилище/raw/01_<проект>/Документы/»). Дальше:

1. Откройте файл правок и пройдитесь сверху вниз.
2. Каждую правку давайте Claude в чате одной командой: «сделай правку 1 из отчёта».
3. После всех правок — запустите промпт повторно и посмотрите новый балл.

Цель — не выйти на 100 за один заход, а понять, **в какой стороне расти**.

---

## Когда запускать проверку повторно

- Сразу после прохождения шагов 1–4 — базовая оценка.
- Через неделю активного использования — после первых 20+ новых файлов.
- Через месяц — должны быть 50+ файлов, заполненный `Заметки_чат.md` в каждом проекте, минимум 5 wiki-страниц.
- Через 3 месяца — целевая точка, должны выйти на 80+ баллов и приближаться к структуре Automy AI.
